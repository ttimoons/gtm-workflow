Project context for AI coding assistants.

## Project Overview

GTM Workflow Sandbox — a visual node-based diagramming tool for planning Google Tag Manager implementations. Users drag-and-drop GTM components onto a React Flow canvas, connect them to map data flow, and save/export architectures.

## Tech Stack

- **Frontend**: React 19, TypeScript 5.9, Vite 7, Tailwind CSS 4, Zustand (state), @xyflow/react (React Flow), Lucide icons
- **Backend** (optional, local-only): Flask, Playwright (headless Chromium for domain scanning)
- **Deployment**: Netlify (frontend only — backend is not deployed)

## Key Patterns

### Node System
All custom nodes extend `BaseNode` which provides: resizable handles via `NodeResizer`, context menu (duplicate/delete), connection handles, and dynamic min-size based on content measurement. Node types are registered in `src/nodes/index.ts`.

### Tag System
Tags are data-driven via `src/data/tagRegistry.tsx`. Each tag type defines its label, icon, brand color, and optional fields. `TagNode` renders generically based on `data.tagType`.

### State Management
Single Zustand store (`useFlowStore`) manages: React Flow nodes/edges, project metadata, save/load, and all mutations. Projects persist to both localStorage (fallback) and server files via the Vite plugin API.

### Project Persistence
- Dev: Vite plugin (`plugins/projectsApi.ts`) serves CRUD API for `public/projects/*.json`
- Prod: Static `manifest.json` (generated by `scripts/sync-projects.js` at build time)
- `public/projects/*.json` is gitignored; `.gitkeep` keeps the directory tracked
- `manifest.json` is auto-synced via npm lifecycle hooks (predev, prebuild, postinstall)

### Versioning
App version is read from `package.json` and injected via Vite's `define` as `__APP_VERSION__`. Displayed in the sidebar footer.

## Build & Dev Commands

```bash
npm run dev          # Vite dev server (auto-syncs project manifest)
npm run build        # TypeScript check + Vite production build
npm run preview      # Preview production build
./setup.sh           # Full install (npm + pip + Playwright Chromium)
./start-dev.sh       # Start both frontend + Flask backend
```

## Common Conventions

- **TypeScript strict mode** — no unused locals/params, no unchecked side-effect imports
- **Tailwind CSS 4** — utility-first, no CSS files
- **No CSS modules or styled-components**
- File naming: PascalCase for components/nodes, camelCase for utilities/stores
- Node data types are defined in `src/store/types.ts` — every node data interface must include `temporary?: boolean`
- All node types must be added to the `AppNode` union in `types.ts` and registered in `nodes/index.ts`
- Tag types must be registered in `tagRegistry.tsx` with icon, label, color
- Canvas sidebar items go in `Sidebar.tsx` using `SidebarItem`
- JSON exports follow the shape `{ id, name, nodes, edges, createdAt, updatedAt }`

## Important Gotchas

- `public/projects/*.json` files are gitignored — don't try to commit them
- The backend is local-only and not deployable to Netlify/Vercel
- When adding new node types: update `types.ts` (data type + AppNode union), `nodes/index.ts` (register), `Canvas.tsx` (minimap colors, onDrop sizing if needed), and `Sidebar.tsx`
- `ZoneNode` uses `zIndex: -1` so it sits behind other nodes on the canvas
- PNG export temporarily overrides the viewport transform — always restore in a `finally` block
